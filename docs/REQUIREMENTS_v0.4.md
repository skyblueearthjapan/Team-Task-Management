### 1.2 社内カレンダー（シート名: `社内カレンダーマスタ`）

| 列 | 型 | 例 | 用途 |
|---|---|---|---|
| `日付` | date | `2026-04-04` | キー |
| `区分` | enum | `休日` / `出勤土曜` / `祝日` | カレンダー描画ロジック |
| `曜日` | string | `土` / `日` / `水` | 表示・整合性確認用 |
| `備考` | string | `昭和の日` 等 | 祝日名 |

- 平日でカレンダーに登録なし＝**通常営業日**
- 「**出勤土曜**」は土曜だが営業日扱い（重要）

### 1.3 社内カレンダーに基づくガント背景色ロジック（v0.4 で確定）

| 状態 | ガントセル背景 | 日付ヘッダ装飾 |
|---|---|---|
| 通常平日 | 白（`--bg-panel`） | 通常 |
| **出勤土曜**（区分=出勤土曜） | 白（**通常営業日扱い**） | 曜日「土」表示・通常背景・小さく「出勤」バッジ |
| 休日（区分=休日） | グレー（`#f0e9d8` weekend色） | 通常の土日色（土=#4a5b6e／日=err） |
| 祝日（区分=祝日） | グレー＋微赤帯 | 日番号 err 色＋備考のツールチップ |
| 本日 | アクセントソフト | 下線2px＋アクセント色 |

> **設計上の含意**: 「土曜＝必ず休み」と決め打ちしない。日付ロジックは曜日ではなく**社内カレンダーの区分**を一次情報とする。

## 2. 工番選択時の自動補完仕様

下部報告フォームで工番をプルダウン選択した瞬間、**3 列**を読み取り専用で自動表示する:

| 表示位置 | 元列 | 表示形式 |
|---|---|---|
| 受注先（カラム） | `受注先` | テキスト（小） |
| 納入先（カラム） | `納入先` | テキスト |
| 品名（カラム） | `品名` | テキスト（強調） |

**任意表示（折りたたみ詳細）**:
- 納入先住所 — マウスオーバーでツールチップ表示
- 数量 — 詳細パネル展開時のみ
- 取込日時 — 管理画面のみ

### 2.1 「（一般作業）」選択時の挙動

工番プルダウンには `（一般作業）` という固定エントリを追加。選択すると自動補完欄は空欄、自由記述のみ入力可能。

## 4. データフロー詳細（v0.4 で具体化）

### 4.1 起動時データロード

```
ブラウザ ─────▶ GAS doGet()
                   │
                   ├─ Staff               (バインド DB)
                   ├─ WorkTypes           (バインド DB)
                   ├─ Settings            (バインド DB)
                   ├─ 工番マスタ全件      (外部 openById)
                   ├─ 社内カレンダー全件  (外部 openById)
                   └─ Schedules + DailyReports（直近の表示範囲分）
                   ▼
                JSON で一括配信（payload size 推定 100〜200KB）
```

### 4.2 マスタキャッシュ更新

外部マスタは**起動時にロード**。リロード（F5）で再取得。明示更新ボタンも提供（要確認）。
