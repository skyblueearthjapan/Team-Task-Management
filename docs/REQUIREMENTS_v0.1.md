# 機械設計技術部 タスク管理アプリ 設計仕様書 v0.1（ドラフト）

> 本書は初期ドラフトです。「11. 確認事項」に未確定事項を列挙しています。合意が取れた項目から順次本文へ反映し、すべて解消した時点で **v1.0** として確定します。

---

## 1. 概要

機械設計技術部のスタッフ約6名のタスクを一元管理する Web アプリケーション。Google スプレッドシートを DB として活用し、Google Apps Script (GAS) で WebApp として提供する。日々の予定（ガントチャート）・作業内容・前日までの作業報告を記録し、必要に応じてローカル PC の Outlook 経由でチーム全員へ自動メール送信する。

参考: 姉妹プロジェクト [Electrical-Work-Scheduling](https://github.com/skyblueearthjapan/Electrical-Work-Scheduling) — **デザインとガント操作感のみ参照**。機能要件は本書を正とする。

## 2. 目的とスコープ

### 2.1 目的

- 機械設計技術部スタッフ全員のスケジュール・作業状況を可視化する
- 日次の作業報告を自動でチーム共有できるようにする
- 工番／製品マスタを再入力させず、選択ベースで素早く記録できるようにする

### 2.2 スコープ

**含む**
- ガントチャートによる週次／月次スケジュール管理
- 日次の「本日の作業内容」「前日までの作業報告」入力
- 工番マスタ・スタッフマスタ・作業内容マスタとの連携
- スタッフ単位での「メール送信」依頼機能（Outlook 経由）

**含まない（今フェーズでは想定しない）**
- 工数集計・原価管理
- 顧客側への直接配信
- モバイル専用 UI（PC ブラウザ前提）

## 3. 利用者

| 区分 | 想定人数 | 操作範囲 |
|---|---|---|
| 機械設計技術部スタッフ | 約6名 | 自分の予定・作業内容・報告を入力／全員分を閲覧 |
| 部内管理者 | 1〜2名 | 上記に加えてマスタ編集・送信履歴閲覧 |

- 全員 LINE WORKS ドメイン (`@lineworks-local.info` 想定) のユーザー。
- WebApp のアクセス制御は「LINE WORKS のユーザー」ドメイン制限で行う。

## 5. 画面構成

画面は単一ページで、上下2エリアの縦分割構成とする。

```
┌──────────────────────────────────────────────────────┐
│ ヘッダー（部署名・週切替・本日ボタン・更新日時）         │
├──────────────────────────────────────────────────────┤
│                                                      │
│ 上部：ガントチャート                                   │
│   行 = スタッフ約6名                                   │
│   列 = 日付（週／2週／月の切替）                        │
│   セル = ドラッグで期間バーを作成・移動・伸縮            │
│                                                      │
├──────────────────────────────────────────────────────┤
│ 下部：作業内容 / 前日までの作業報告                     │
│   スタッフ別タブ または 横並びカラム                    │
│   1人あたり最大 5〜6 行                                │
│   各行: 日付期間 / 工番 / 納入先 / 製品名 / 作業内容    │
│        / 自由記述 / [メール送信]                       │
└──────────────────────────────────────────────────────┘
```

### 5.1 上部：ガントチャート

- **行**: 機械設計技術部スタッフ約6名（スタッフマスタから取得）
- **列**: 日付軸。表示期間は 1週間 / 2週間 / 1ヶ月 を切替（既定: 1週間）
- **操作**:
  - セル横方向ドラッグでバー作成（期間設定）
  - バー両端ドラッグで期間延長／短縮
  - バー本体ドラッグで日程移動
  - ダブルクリックで詳細編集ダイアログ（工番・作業内容・備考）
- **同一スタッフ内の重複**: 自動レーン分割で縦に積む（参考リポと同じ挙動）
- **休日**: 土日／祝日はグレーアウト（祝日マスタの扱いは §11 で確認）
- **保存**: 楽観的UI + 数秒デバウンスで自動保存（保存中インジケータ表示）

### 5.2 下部：作業内容・報告欄

- 表示単位: **日付ごとに「何日から何日まで」**の期間表示。同一作業が複数日にまたがる場合は1行にまとめる。
- スタッフ別に最大 **5〜6項目**まで入力可能。
- 各行の入力フィールド:

| 項目 | 入力方法 | 備考 |
|---|---|---|
| 期間 | 日付ピッカー or ガントから自動連携 | §5.1 で設定したバーが右隣に「予定」として表示される |
| 工番 | プルダウン（工番マスタ） | 一般作業の場合は「（一般作業）」を選択 |
| 納入先 | 自動入力 | 工番マスタから引当 |
| 製品名 | 自動入力 | 工番マスタから引当 |
| 作業内容 | プルダウン（作業内容マスタ） | 多項目選択を許可するかは §11 で確認 |
| 詳細／自由記述 | テキスト | 任意 |
| 区分 | 「本日の作業内容」 / 「前日までの作業報告」 | 行ごとに選択 |

- ガントチャートで設定した期間は、対応する報告行の右側に**予定バー**として表示する。

## 7. 機能仕様

### 7.1 ガント操作（上部）

- ドラッグでの期間作成・移動・伸縮（参考リポ準拠）
- 楽観的 UI 更新 → 数秒デバウンス → サーバ保存
- 保存失敗時はバーを赤枠化＋トースト通知＋ LocalStorage バックアップ
- 表示期間切替（週／2週／月）
- 「本日」ジャンプ
- 凡例: 工番別の色分け or スタッフ別の色分け（要確認）

### 7.2 作業内容・報告入力（下部）

- スタッフ単位で 5〜6 行のグリッド表示
- 工番プルダウンを選んだ瞬間、納入先・製品名が自動補完
- 「（一般作業）」選択時は工番列は空のままで自由記述のみ
- 「本日／前日」セクションのトグル
- 上部ガントの該当バーに対応する行は、右隣に**予定期間**を視覚バーで表示

### 7.3 マスタ連携

- 起動時に Staff / WorkTypes / Holidays をフロントへ一括ロード
- Koban / Products は件数次第でキャッシュ戦略を変える（§11）
- マスタ更新は管理者専用 GAS 関数で行い、UI からは編集不可（要確認）

### 7.4 メール送信機能

#### 7.4.1 UI

- 各スタッフ行の右端に **「メール送信」ボタン**
- 押下時の確認ダイアログ:
  - 宛先（Staff マスタの全 active 者のメール）
  - 件名プレビュー
  - 本文プレビュー（本日の作業内容＋前日までの作業報告）
- 確定で `MailQueue` へ `status=pending` でレコード追加

#### 7.4.2 EXE 連携フロー

```
[GAS UI]                [Spreadsheet DB]              [ローカル EXE]              [Outlook]
   │                          │                             │                       │
   │ ボタン押下                │                             │                       │
   ├─ MailQueue insert ──────▶│                             │                       │
   │                          │     30〜60秒ごとに           │                       │
   │                          │◀─── pending を取得 ──────────┤                       │
   │                          │  status=picked 更新 ◀──────┤                       │
   │                          │                             │── COM で送信 ───────▶│
   │                          │  status=sent / failed ◀────┤                       │
   │ 履歴更新                  │                             │                       │
   │◀── ポーリング or 手動更新 ─┤                             │                       │
```

- 排他制御: `pending → picked` への更新は EXE が **行ロック相当**で行う（同 ID を複数 EXE が掴まないよう、`pickedBy` と `pickedAt` 列でアトミックに更新）
- リトライ: `failed` から N 分後に再試行可。上限超過で `dead` へ
- ログ: 成功・失敗の双方を `MailLog` に追記

#### 7.4.3 メール本文フォーマット（案）

```
件名: 【機械設計技術部】{スタッフ名} 業務報告 {YYYY/MM/DD}

{スタッフ名} です。本日の業務をご報告いたします。

▼ 本日の作業内容
1. {期間} {工番} {納入先} {製品名} / {作業内容} {詳細}
2. ...

▼ 前日までの作業報告
1. {期間} {工番} {納入先} {製品名} / {作業内容} {詳細}
2. ...

以上、よろしくお願いいたします。
```

> 件名・本文の最終フォーマットは §11 で確認。

## 8. 外部連携

### 8.1 ローカル送信エージェント (EXE)

- **役割**: スプレッドシートをポーリングし、`MailQueue.status=pending` を検知したら Outlook へ送信を依頼
- **動作環境**: 担当者の業務 PC（Outlook がインストール済み・ログイン済み）
- **認証**: GAS WebApp 経由 or Sheets API（OAuth サービスアカウント）— §11 で確認
- **常駐方法**: タスクスケジューラ または スタートアップ — §11 で確認
- **ヘルスチェック**: 定期的に `Settings` シートへ `lastHeartbeat` を書き込み、UI に「エージェント生存中」の表示を出す（推奨）

### 8.2 工番マスタ別ブックの参照

- 別ブック ID は Script Properties に保管
- 読み取り権限のあるアカウントでアクセス
- 同期方式: 起動時取得 or 1日1回キャッシュ更新（§11）

## 9. 非機能要件

| 区分 | 要件（案） |
|---|---|
| パフォーマンス | 操作レスポンス 200ms 以内（楽観的UI）／保存反映 5 秒以内 |
| 可用性 | GAS の SLA に準拠 |
| データ保持 | Schedules / DailyReports は無期限／MailQueue は 90 日後アーカイブ（要確認） |
| セキュリティ | LINE WORKS ドメイン制限／Script Properties で機密値管理 |
| 監査 | 主要シートに `updatedAt` `updatedBy` を持たせる |
| バックアップ | スプレッドシート自動版＋日次でアーカイブシートへコピー（要確認） |

## 10. 参考プロジェクトとの関係

- **デザイン**: 参考リポのダークテーマ・タブ構成・ガントの操作感を流用
- **ファイル構成**: `Code.gs / Config.gs / DataService.gs / SetupService.gs` の役割分担を踏襲
- **異なる点**:
  - Calendar 双方向同期は不要（タスク管理に振る）
  - 代わりに **MailQueueService** とローカル EXE 連携を新設
  - 業者カレンダー6社のサブタブは無し → スタッフ6名のガント1枚に置換
