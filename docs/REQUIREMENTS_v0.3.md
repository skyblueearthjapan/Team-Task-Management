## 1. 概要

機械設計技術部のスタッフ約6名のタスクを一元管理する Web アプリケーション。Google スプレッドシート（バインドDB）+ 外部マスタスプレッドシート（工番／社内カレンダー）の二段構成。GAS WebApp で UI を提供し、ローカル PC 常駐 Python EXE が Outlook 連携を担う。

## 4. アーキテクチャ確定事項

| 項目 | 確定内容 |
|---|---|
| EXE の配置 | **各 PC に1台ずつ配置**（送信専用機ではなく分散） |
| EXE → スプレッドシート の経路 | **EXE → HTTP → GAS WebApp 専用エンドポイント → Sheets** |
| EXE 認証 | 各 EXE が `Settings.EXE_API_TOKEN` をヘッダで送信し、WebApp 側で照合 |
| Outlook 送信元 | **各 PC の Outlook 既定プロファイル** から送信 |
| 送信モード | スプレッドシート上で `mode=draft` / `mode=send` を**全員が選択可能** |
| ポーリング間隔 | **30 秒**（負荷次第で 60 秒へ調整可） |
| 件名・本文テンプレ | **EXE 内ハードコード**（v0.2 §7.4.6 のドラフトで承認） |
