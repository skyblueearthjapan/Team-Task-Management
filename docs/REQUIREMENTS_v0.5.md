# 機械設計技術部 タスク管理アプリ 設計仕様書 v0.5（ドラフト）

> 本書は v0.4 を更新したドラフトです。マスタ周辺の細部仕様を確定させ、v1.0 に向けた残課題を v0.3 §9 の C〜H に絞り込みました。
>
> **v0.4 → v0.5 主要変更点**
> - **納入先住所** → ツールチップ表示（マウスホバー時）
> - **祝日名** → ツールチップ表示（マウスホバー時）
> - **WorkTypes カテゴリ分類は廃止** → フラット13項目構成へ確定
> - **作業内容は単一選択**（1行=1作業）
> - 工番マスタの「数量・取込日時」は UI に表示しない

---

## 1. 確定した仕様（v0.5 反映分）

### 1.1 工番プルダウンと自動補完

| 仕様 | 内容 |
|---|---|
| プルダウン表示形式 | `LW23012  住友建機㈱  4ton応用機アタッチメントポジショナー`（工番＋受注先＋品名） |
| 検索方式 | インクリメンタル検索（844 行をフロントで部分一致フィルタ） |
| 自動補完される列 | 受注先 / 納入先 / 品名 |
| **納入先住所** | **ツールチップ表示**（行にマウスホバー時に吹き出しで表示） |
| 数量・取込日時 | UI に表示しない（管理画面のみ） |
| 「（一般作業）」エントリ | プルダウン先頭に固定。選択時は補完欄空欄＋自由記述のみ |

### 1.2 社内カレンダーの反映ルール

| 区分 | 営業日扱い | ガント表示 |
|---|---|---|
| 通常平日 | ◯ | 白背景 |
| **出勤土曜** | **◯** | 白背景＋ヘッダに「**出勤**」バッジ表示 |
| 休日（土／日） | × | グレー背景 |
| 祝日 | × | グレー背景＋日番号エラー色 |
| 未登録の土日 | × | 通常休日扱い（カレンダーに無くても土日は休日） |
| **祝日名（備考）** | – | **ツールチップ表示**（日付セルにホバー時、例「昭和の日」） |

> 一次情報は社内カレンダーマスタの**区分**。曜日では休日判定しない。

### 1.4 シート定義の更新

#### `WorkTypes` シート（v0.5 簡略化）
```
列: id, name, displayOrder, active
```

#### `Schedules` シート（変更なし）
```
列: id, staffId, startDate, endDate, kobanCode, workTypeId, note, lane, createdAt, updatedAt
```

#### `DailyReports` シート（変更なし）
```
列: id, staffId, reportDate, section, seq, periodStart, periodEnd,
    kobanCode, workTypeId, detail, linkedScheduleId, updatedAt
```

> 工番情報（受注先・納入先・品名）は実行時に外部マスタから引当るため、`DailyReports` には `kobanCode` のみ保持する（非正規化キャッシュは持たない）。

## 3. 残課題（v1.0 確定までに必要）

v0.3 §9 の C〜H を、優先度順に再掲します。

### C. ガントチャート詳細（中優先）
1. 期間粒度: **日単位**で確定で良いですか？（時間単位や半日単位は不要？）
2. 連続スクロールの**初期表示位置**: 「今日」を中央／月初／週初、どれが好ましいですか？
3. 描画範囲の既定: **過去30日 / 未来90日** で良いですか？
4. **色分け基準**: 「工番別」「作業内容別」「スタッフ別」のどれが良いですか？  
   → デザインモック (`docs/wireframe.html`) は**工番別**で組んでいます

### D. 日報入力（中優先）
5. 「本日／前日まで」セクションの**保存単位**は日次レコードで良いですか？
6. ガントで作成した予定を、**自動的に下部の報告行へ初期表示**したいですか？  
   案: 既定 ON。手動でも追加・編集・削除可
7. 自由記述の**文字数上限**の目安（例: 200 文字 / 500 文字）

### E. メール送信運用（中優先）
8. 送付先は **Staff マスタ active 全員**で確定で良いですか？
9. **CC** や **BCC** の固定運用は必要ですか？（例: 部長宛 CC 固定 / 本人 BCC）
10. 1 日に同じスタッフから**複数回送信**を許可しますか？（または 1 日 1 回までに制限？）
11. 送信失敗時の**通知先**: 管理者宛メール / UI のバッジのみ / 両方？

### F. ローカル EXE 運用（高優先）
12. EXE が**応答していない**ことの通知方法:  
    案: GAS UI のヘッダに「最終ハートビート N 分前」表示 ＋ 5 分以上応答無しで赤バッジ
13. EXE の**配布方法**: 共有フォルダにビルド済み EXE を配置／インストーラ作成／手動コピー
14. EXE 内テンプレート変更時の**再配布フロー**: 私がビルドして共有 → 各 PC で上書きコピー

### G. データ保持（低優先）
15. `Schedules` / `DailyReports` の保持期間（例: 1 年 / 無期限）
16. `MailQueue` 完了レコードの保持期間（例: 90 日）

### H. デプロイ・運用（低優先）
17. clasp によるソース管理は技術部側で運用しますか？（GitHub への push 含む）
18. ステージング GAS プロジェクトを別建てしますか？
